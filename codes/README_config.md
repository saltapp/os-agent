# UI-TARS 配置文件说明

本项目使用以下配置文件来方便修改系统行为:

## 系统提示词 (`system_prompt.md`)

此文件包含发送给模型的系统提示，定义了GUI代理的行为和可用操作。
修改此文件可以调整模型的响应方式和对任务的理解能力。

主要部分:
- 输出格式定义
- 可用操作空间
- 使用说明和注意事项

## 用户提示词 (`user_prompt.md`)

此文件包含需要执行的具体任务描述。
每次修改后，程序会自动加载最新的任务描述。

## 使用方法

1. 如需修改系统行为，编辑 `system_prompt.md` 文件
2. 如需修改执行任务，编辑 `user_prompt.md` 文件
3. 使用默认设置运行程序: `python chat_with_model_ui_tars3.py`

## 操作控制

- **ESC键**: 在任务执行过程中，随时按下ESC键可以中断执行流程
- 程序会在当前操作完成后检测中断信号并安全退出
- 中断后会保存已执行步骤的历史记录

## 性能优化

- **执行速度**: 通过修改代码中的`execution_speed`变量可调整任务执行速度
  - 默认值为0.5秒，值越小执行越快
  - 建议范围: 0.2(快速) ~ 1.0(稳定)
  - 过低的值可能导致操作不稳定
- **优化措施**:
  - 使用客户端缓存减少API连接开销
  - 自适应的等待时间根据操作类型调整
  - 动态调整鼠标和键盘事件的速度

## 注意事项

- 修改提示词时请保持格式一致
- 如果读取文件失败，程序会使用内置的默认值
- 编辑文件时请使用UTF-8编码以支持中文字符
- 首次运行可能需要安装keyboard库: `pip install keyboard`

# 配置说明

为了保护您的API密钥和其他敏感信息，本项目使用配置文件来存储这些信息，而不是直接写入代码中。这样可以防止敏感信息被上传到GitHub等公共代码仓库。

## 配置文件设置

1. 在`codes`目录下，您会发现一个名为`config.template.json`的模板文件
2. 首次运行程序时，系统会自动基于此模板创建`config.json`文件
3. 您需要编辑`config.json`文件，填入您的实际API密钥和其他信息

## 配置项说明

```json
{
  "api": {
    "key": "your_api_key_here",     // 您的API密钥
    "base_url": "https://ark.cn-beijing.volces.com/api/v3",  // API基础URL
    "model_id": "your_model_id_here"  // 使用的模型ID
  }
}
```

## 注意事项

- **请勿**将包含实际API密钥的`config.json`文件上传到GitHub或其他公共仓库
- 项目的`.gitignore`已配置为忽略`config.json`文件
- 如果您需要与他人共享代码，请只分享`config.template.json`模板，而不是您的实际配置文件

## 如何获取API密钥和模型ID

1. 登录到您的API提供商后台
2. 创建或查看您的API密钥
3. 查询可用的模型ID
4. 将这些信息填入`config.json`文件中
